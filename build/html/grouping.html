ncobj.grouping.check_group_name_clashes(group)¶

Check this group and subgroups for any name clashes between components.

If found, raise a NameConflictError describing the first clash.

Note

Name collisions can occur between variables, subgroups and user-types: In NetCDF, these components share a namespace within each group.

ncobj.grouping.add_missing_dims(group)¶

Create new definitions for any missing dimensions in the group.

A missing dimension is one referred to by a variable in ‘group’ or its subgroups, for which no definition can be located by find_named_definition(). The new ones are created in ‘group’.

Returns:
A list of the definitions created for missing dimensions.

Note

If ‘group’ is not itself the root, then a matching definition may be found in a parent group. In these cases, no new definition is created (even though the required definition is outside ‘group’).

ncobj.grouping.has_no_missing_dims(group, fail_if_not=False)¶

Check that matching definitions exist for all dimensions used in the variables of this group (and its sub-groups).

Kwargs:

  • fail_if_not (bool):
    If set, then if and when a missing dimension is found, raise an IncompleteStructureError, instead of just returning False.

Note

If ‘group’ is not itself the root, then a matching definition may be found in a parent group. Such a dimension is not counted as ‘missing’ (even though the required definition is outside ‘group’).

ncobj.grouping.check_consistent_dims_usage(group)¶

Check that the requirements for all dimensions are consistent, and if not raise a DimensionConflictError.

This means that all references to each dimension must have the same length. Where variables have attached data, the length is taken from the data shape instead of the attached Dimension object, and the number of dimensions must also match. Each dimension must also have a known length, meaning that at least one reference must define the length, or have attached data.

Note

Can only be used on groups with no missing dimensions, as described for has_no_missing_dimensions(). Otherwise a IncompleteStructureError will be raised.

ncobj.grouping.complete(group)¶

Make this group internally consistent, by adding any missing dimension definitions and linking all variable dimensions to their definitions.

This makes the structure fully compliant with NetCDF constraints. This ensures it is suitable to be written to a file.

Dimension definitions are made consistent with the data and dimension information of all variables that reference them. If this is not possible, as decribed for check_consistent_dims_usage(), a DimensionConflictError is raised. A dimension definition will also be made ‘unlimited’ if any of the references requires it.

Note

A NameConflictError can also result if components have conflicting names, as described for check_group_name_clashes().
